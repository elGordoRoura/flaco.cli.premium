### r/LocalLLaMA

**Title:** Built agent swarms with Ollama - 10 specialized coding agents collaborating locally

**Body:**
```
Implemented multi-agent collaboration in Flaco AI PREMIUM using Ollama as the backend.

**The Setup:**

10 specialized agents, each with different:
- System prompts (domain expertise)
- Thinking patterns
- Response styles
- Task routing keywords

All running on local Ollama instance.

**How Agent Swarms Work:**

1. **Task Analysis**: Parse user request for complexity signals
2. **Agent Selection**: Pick 2-4 relevant agents based on domains
3. **Task Decomposition**: Break into parallel sub-tasks
4. **Parallel Execution**: Each agent handles their specialty
5. **Result Synthesis**: Combine outputs into coherent solution

**Example:**

Request: "Optimize this API for performance"

Swarm: Eddie (APIs) + Craig (Database) + Phil (Performance)

Each works on their aspect:
- Eddie: API design, caching, rate limiting
- Craig: Query optimization, indexing
- Phil: Profiling, memory optimization

Results merged into comprehensive solution.

**Challenges:**

1. **Context Management**: Each agent needs sufficient context
2. **Result Conflicts**: Sometimes agents suggest conflicting approaches
3. **Performance**: Multiple LLM calls can be slow (mitigated with smaller models)

**Solutions:**

- Shared context store
- Consensus algorithm for conflicts
- Parallel execution where possible
- 7B models for speed, 32B for quality

**Models Tested:**

Works best with:
- qwen2.5-coder:32b (best quality)
- qwen2.5-coder:7b (best speed)
- codellama:13b (good balance)

**Performance:**

Complex tasks: 3-5x faster than sequential
Simple tasks: Single agent (no swarm overhead)

**Open Source:**

GitHub: https://github.com/RouraIO/flaco.cli.premium

Would love to discuss optimization strategies with this community!

Specific questions:
1. Best models for agent collaboration?
2. Prompt engineering tips for specialized agents?
3. Better task decomposition approaches?
```
